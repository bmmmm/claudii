## [Unreleased]

### Added
- `on`/`off`: touch/remove `presence.file` when configured (`CLAUDE_CLIENT_PRESENCE_FILE` suppresses Claude Code mobile notifications)
- **The Account section shows each rate limit's wall-clock reset time** ‚Äî the bare `claudii` overview already printed the relative reset countdown (`‚Ü∫3h45m`); it now also pins the absolute time the window resets, so you can plan against a clock instead of doing the arithmetic: `‚Ü∫3h45m (16:30)` for the 5h limit and `‚Ü∫2d4h (Wed 16:30)` for the 7d limit (the weekday is unambiguous within a week, so no full date). The reset epochs were already cached (`reset_5h`/`reset_7d`, lifted from the Claude Code rate-limit payload by `claudii-cc-statusline`); rendering reuses the timezone-aware `_fmt_abs` (honors `display.timezone`) and only appears when a fresh `session-*` cache carries the reset timestamp ‚Äî no new data path.

### Changed
- **Bare `claudii vibemap` now defaults to the 30-day strip** ‚Äî the default view (no subcommand) was the 8√ó7 weekday √ó 3-hour grid, which aggregates the calendar away and shows no date range. `claudii vibemap` now renders the last 30 days √ó 24 hours (one row per day, hour resolution), so the actual month of activity reads at a glance. The weekday grid stays available as `claudii vibemap grid`; `claudii vibemap strip` and a bare `--days` now default to 30 (was 14), range 1‚Äì90 unchanged. Reuses the existing `vibemap-strip.awk` aggregator (already windowed by `maxdays`) ‚Äî no new renderer, no schema change. The bare-overview Activity mini-strip follows the same 30-day window (was 43d), so the strip and the `claudii vibemap` view its hint points to now agree on one window.
- **Bare `claudii` overview is ~2.2√ó faster (1.55s ‚Üí 0.68s here)** ‚Äî the overview had grown four heavy, serialized subprocess hotspots that together were essentially the entire runtime. Fixed with no output change (byte-identical account/usage figures verified; 1443 tests green under `/bin/bash` 3.2):
  - **History files are mtime-gated to the window.** The 30-day usage sparkline parsed *every* history file (163k rows here) including the frozen legacy `history.tsv` and months that lie entirely outside the window ‚Äî `usage_spark.awk` discarded those rows one by one via its per-row guard. `_collect_history_files` gained an optional mtime cutoff: an append-only file whose newest row (== mtime) predates the window holds no in-window rows and is skipped (cost/trends pass no cutoff, unchanged).
  - **One history pass, not two.** The account line's "today tokens/sessions" and the usage sparkline each ran a full per-session attribution scan over the same data. They are now a single `usage_spark.awk` pass driven from `_ov_gather` (the awk emits `today_sessions` as a 6th summary field); `_ov_render_usage` formats the result instead of re-scanning.
  - **The two heaviest subprocesses run concurrently.** `claude agents --json` (Node startup, ~0.37s) for the live-agent PID list and the merged history awk (~0.5s) need nothing from each other, yet ran back to back. They are now kicked into the background before rendering and reaped inside `_ov_gather` ‚Äî the agents fetch before the session-cache loop (which needs it), the history pass after (so the awk overlaps the loop and the fetch). Best-effort, with a synchronous fallback; safe to background in `bin/` (non-interactive, no job control), temp files removed on reap.
  - **The vibemap mini-strip cache is warmed in parallel too**, so the activity section's read is a cheap `cat` even when its 60s TTL cache has expired (the common case for occasional overview use) ‚Äî the cold path dropped from 0.89s to 0.68s, matching the warm path.

## [v0.23.1] ‚Äî 2026-06-15

### Changed
- **CC-Statusline hotpath forks trimmed** ‚Äî the in-session statusline reruns on every render (frequent in a live session), so each per-render subprocess spawn is pure overhead. Three cuts, no output change: the eight `_cache_get` lookups that read the already-loaded session cache no longer each run in a `$(...)` subshell (they write a global `_CG`, the same fork-free pattern `_tok`/`_dur` already use); the ClaudeStatus age display and the self-refresh TTL check now share one `stat` on `status-models` instead of forking it twice; and burn-ETA + pace (same `dur`/`rate` inputs) are emitted by a single `awk` instead of two. ~10 `fork()`s removed per render on the warm path. Verified byte-identical cache writes, green under `/bin/bash` 3.2.
- **`claudii cache` and `tools` render without per-row `awk` forks** ‚Äî both formatted their percentages (cache hit rate, per-tool error rate) with an `awk` fork *per table row*, so a 30-day `cache` window spawned ~30 awks only to print `"%.1f"`. The percentages are now computed and formatted to one decimal inside the aggregating `jq` (a `pct1s` def); bash only renders. `cache` additionally folds its three separate `jq` passes (per-day, per-model, summary) into one tab-delimited multi-section stream read once (collect-then-render, set-u-safe under bash 3.2). No output change ‚Äî byte-identical, 1443 tests green under `/bin/bash` 3.2.
- **`claudii tokens` and `session` each run one `jq` instead of three / seven** ‚Äî both re-queried the same in-memory aggregate (`tokens`: by-type, by-model, by-day; `session`: meta, dominant-model, tokens, tools, subagents, stop-reasons, limit-hits) with a separate `jq` fork per section. Each now does a single tagged, US-delimited pass that bash reads once into arrays (collect-then-render). `tokens` also computes its shared day-filter once instead of three times. No output change ‚Äî byte-identical (including the empty-`first_seen` no-shift guard), 1443 tests green under `/bin/bash` 3.2.

## [v0.23.0] ‚Äî 2026-06-15

### Added
- **`claudii tokens`, `tools`, `limits` and `cache` gained cycleable named windows** ‚Äî the rolling window can now be switched without remembering `--days`: a named token (`today`/`day`, `week`, `month`, `quarter`, `year`) or a generic `<N>d` (e.g. `claudii tokens 14d`) resolves to a day count, alongside the existing `--days N`/`-d N`. One shared `_insights_window` helper parses and validates the window for all four commands (replacing four duplicated `--days` loops), so the named windows, the `<N>d` form, the positive-integer guard, and an actionable unknown-token error (`unknown tokens argument: lastweek` + a `today|7d|30d|year` hint) stay identical across the class. A 1-day window now reads `today` in the section header instead of the ungrammatical "last 1 days" (shared `_insights_window_label`). The command name is sliced off with `"${@:2}"` at the call site (the dispatcher passes the full argv), matching `skills-cost`/`session`.
- **`claudii tokens`, `tools`, `limits` and `cache` gained `--json`** ‚Äî previously these four insight views silently swallowed the global `--json`/`--tsv` flags (`bin/claudii` strips them into `$_FORMAT`, but the commands never read it). Each now emits the same curated aggregate the pretty path renders, as structured JSON: `tokens` ‚Üí `by_type`/`by_model`/`by_day` with derived `hit_pct`; `cache` ‚Üí `per_day`/`per_model`/`summary`; `tools` ‚Üí `total_calls`/`ok_pct`/`tools[]`/`subagents[]`/`thinking_blocks`; `limits` ‚Üí `total`/`hits[]` (newest-first)/`by_model[]`. Raw model ids (not the friendly labels, which are a bash-render concern), `error_pct`/`hit_pct` rounded to one decimal. The empty-cache case emits the same envelope shape with empty arrays (the builders run with `merged="{}"`), so a consumer never has to special-case "no data". `--tsv` is rejected with a pointer to `--json` rather than silently ignored ‚Äî these views are nested and a flat TSV would have to invent a shape.
- **`claudii` overview gained a `usage` section** ‚Äî a 30-day token-per-day sparkline built from the cost/trends history (not the short-lived insights cache, so the trend reaches back a full month, unlike `claudii tokens`). Eight-level Unicode ticks (‚ñÅ..‚ñà) normalized to the busiest day, one tick per local calendar day oldest‚Üítoday, with a dim `today ¬∑ busy-day avg ¬∑ peak` context line (average over *active* days, not √∑30, so idle days do not understate the working rate). Placed right after `account` so the consumption trend is visible at a glance ‚Äî the account line shows today's total, the sparkline shows where today sits in the month (the usage `today` mirrors the account figure, except on the 1st of a month for a session spanning the boundary ‚Äî usage scans all history, account only the current month). Shared per-session `attr_delta` attribution (same heuristic as cost/trends), so a long-running session is not counted cumulatively. New `lib/usage_spark.awk` (daily aggregation) + `_sparkline()` in `lib/render.sh`; config-driven via `overview.sections` like the other sections (shipped defaults gained `usage` after `account` ‚Äî a user config that pins an explicit `overview.sections` keeps its current set until they add it).
- **`claudii cost --forecast`** ‚Äî burn-rate projection, the last view of the token-focus shift (it lives under `cost` because the month half is dollar detail). The **5h budget** block is the token-side forecast: current account-wide rate-limit usage, the burn rate as the slope of `rate_5h` over the last 20 minutes (with an accelerating/slowing note from comparing the two 10-minute halves), a projected time-to-limit shown only when the budget is on track to be hit *before* the window resets (otherwise "resets before limit"), and the reset countdown. The live 5h state (used % + reset epoch) comes from the newest fresh `session-*` cache ‚Äî history carries no reset timestamp, and an expired reset means the cache predates a reset so its rate is stale; with no live session the block degrades and only the month projection renders. The **This month** block projects spend from the same per-session cost deltas `cost` aggregates: spend so far, $/day pace, a month-end projection with an honest `¬±` band = daily-spend stddev √ó ‚àö(remaining days) (random-walk style, so it widens with volatility and collapses near month-end; suppressed as "early ‚Äî wide uncertainty" on day 1), and a vs-last-month verdict. Burn slope needs ‚â•2 samples spanning ‚â•3 min within the current cycle, else it reports "not enough recent samples (idle)" rather than inventing a number. New `lib/forecast.awk` (composed after `epoch_to_date`/`attribution`/`fmt` via `-f`); honors `--json` (structured `five_hour`/`month`) and `--tsv`.
- **`claudii tools`** and **`claudii limits`** ‚Äî two token/efficiency lenses over the rolling window (insights cache). `tools` is the throughput/reliability companion to the cost-focused `skills-cost`: a table of the top tools by call count with share bars and per-tool error rate (a `‚öÝ` flags rates above 5%), the subagent types spawned, total thinking-block volume, and a one-line callout of the highest-error-rate tool (even one hidden in the folded long tail ‚Äî MCP tool names are shortened to their tool segment, the rest folded into `+N more`). `limits` surfaces the previously-unshown `limit_hits[]`: each hit (date, time, model) newest-first, an hour strip showing when hits cluster, and a per-model tally ‚Äî time and model only, since the data carries no percentage and the 5h budget is account-wide (the model is "what ran", not "what is to blame"). Both honor `--days N`; both run under `set -euo pipefail`, so the hour-bucket array is incremented set-u-safely (an unset element trips nounset).
- **`claudii session <id>`** ‚Äî per-session drilldown from the insights cache, matched by session-id substring (first wins, like `pin`). Shows the token split (input / output / cache write as share bars over the real work, plus cache read with its hit rate and the thinking-block count), the top tools with per-tool error markers, and compact lines for subagents, stop reasons, and rate-limit hits (account-wide). Live context %, model, and project come from the `session-*` cache when one still exists; ended sessions (whose `session-*` cache was GC-ed but whose insights cache persists) degrade gracefully ‚Äî the model is inferred from the dominant token bucket, the context bar is dropped, and the project falls back to the repo basename from the transcript path. Shared `_iso_epoch` (ISO-8601 ‚Üí epoch, BSD+GNU) added to `lib/timefmt.sh` for the duration and limit-hit timestamps; `limit_hits` are displayed time + model only (the data carries no %, and the 5h budget is account-wide ‚Äî the model is "what ran", not "what is to blame").
- **`claudii tokens`** ‚Äî token-usage breakdown for the rolling window (default 7 days), the first new view of the token-focus shift. Three sections from the insights cache: **By type** (throughput split across cache read / input / cache write / output as amount-sorted share bars), **By model** (a `‚îÇ`/`‚îº` matrix of input / output / cache rd / cache wr / hit ‚Äî per-model output comes from the `days{}` key-split `YYYY-MM-DD|model`, which `models{}` alone lacks, so no schema bump), and **By day** (in+output per calendar day, bars normalized to the busiest day, with the day-level hit rate). All three are floored to exactly `N` calendar days (the merge windows by `last_seen`, which otherwise drags older day-entries of long sessions into a "7-day" view and makes the bare weekday labels repeat). The cache-hit rate is `cache_read / (cache_read + cache_create + input)` ‚Äî the naive `cache_read / (cache_read + input)` pins at ~100% because cache_read dwarfs raw input. First consumer of the shared bash renderers (`_render_bar_row`, `_render_dgrid`, `_bar_c`, `_render_shead`, `_rep`) now finalized in `lib/render.sh`; `--days N` overrides the window.
- **`skills-cost --compare BEFORE:AFTER`** ‚Äî trend comparison of two time windows (prior `[now-(B+A), now-A)` vs recent `[now-A, now]`, e.g. `--compare 30:30`) so session-close Phase 2.7 can ask "did the SKILL.md edit help?". The headline metric is **out-tokens/call**: output tokens don't scale with session context size, so unlike `$/call` (structurally higher for session-end skills) it isolates whether a skill talks less after an edit. `$/call` is still shown but flagged context-confounded. Needs a bounded merge window ‚Äî `claudii-insights merge` gained `--until-days M` (upper bound `last_seen < now-M`); `_insights_merged_json` takes an optional second arg. `--json` emits joined rows + window/metric metadata.
- **`skills-cost` prices each skill at real per-model rates** (insights schema v5). `attribution_models` went from a bare per-`kind|name|model` call count to a `{calls, in_tok, out_tok, cache_read, cache_create}` object, so `claudii skills-cost` now prices every per-model token bucket at its own tier (Opus $5/$25/M, Sonnet $3/$15/M, Haiku $1/$5/M, Fable $10/$50/M; cache_read 0.1√ó, cache_create 1.25√ó input) instead of a flat Sonnet rate across all models. This un-distorts the cross-skill **ranking** the 2√ó-median outlier rule (and session-close Phase 2.7) compares over ‚Äî a Haiku-dominant skill was ~3√ó overcounted, an Opus-dominant one ~5√ó undercounted. The merge tolerates the pre-v5 scalar shape (orphaned v4 caches that can never re-aggregate keep contributing their call counts), and tokens those caches lack a per-model split for are priced at the flat Sonnet rate as a residual ‚Äî so old data degrades gracefully rather than dropping out of the dollar totals. `skills-cost --json` rows[] contract is unchanged (`tot_usd` is just more accurate); `meta.pricing` documents the per-model rates + the residual fallback. `bin/claudii-insights` `SCHEMA_VERSION` bumped to 5 (rebuild gate); the only hardcoded rates now live in the `_rates` table in `lib/cmd/skills-cost.sh`.
- **Status transition log** ‚Äî `bin/claudii-status` appends every model state change (e.g. `opus ok‚Üídegraded`) to `~/.cache/claudii/status-history.tsv` (epoch, model, old, new; trimmed to 500 lines past 1000). `claudii status` shows the newest 5 under "Recent changes" ‚Äî answers "why was Opus amber 10 minutes ago?" after the cache has already recovered.
- **`claudii status --history [--days N]`** ‚Äî prints the full transition log (newest first) instead of just the last 5; `--days N` limits it to a rolling window. Timestamps honor `display.timezone`; supports `--json` (array of `{ts,model,from,to}`).
- **`display.timezone` config key** ‚Äî IANA timezone (e.g. `Europe/Berlin`) for all absolute timestamps; empty (default) = system local. Applied via the new shared `_fmt_abs` formatter (`lib/timefmt.sh`, BSD+GNU date) to the incident timeline and the transition log in `claudii status`; incident `created_at` is now converted from raw ISO-UTC to the configured zone with a `%Z` suffix.
- **Subagent costs are attributed to the spawning skill** (insights schema v4). Subagent transcripts (`<session>/subagents/agent-*.jsonl`) were invisible to the aggregator (`find -maxdepth 3` never reached them) ‚Äî skill costs only counted main-thread messages. `lib/insights.jq` now ingests them after the parent file and chains `Agent` tool_use ‚Üí `toolUseResult.agentId` ‚Üí the skill active at spawn time, so e.g. `/orchestrate` waves carry their agents' real cost (on live data: orchestrate 380 ‚Üí 1568 attributed calls). Staleness checks and the marker-based scan in `bin/claudii-insights` consider subagent file mtimes too.
- **`skills-cost` model column shows the dominant model** instead of a hardcoded `mixed`. The new flat `attribution_models` map (`kind|name|model` ‚Üí calls) tracks which model served each attributed message; a row shows the friendly label when one model covers ‚â•80% of its calls, otherwise `mixed`.
- **`skills-cost --mcp`** ‚Äî per-MCP-tool cost table from the new `attribution_mcp` block. A message's usage is split evenly across the MCP tools it invokes (totals stay comparable with the skills table); the name column widens and drops the `mcp__` prefix for display.
- **`skills-cost --json` rows carry the token split** (`in_tok`, `out_tok`, `cache_read`, `cache_create`) ‚Äî the judgment signal consumers (session-close Phase 2.7) need: out-heavy means "skill talks too much" (a reply-cap in SKILL.md helps), cache_read-heavy means "runs in fat sessions" (SKILL.md edits won't move the needle). Live proof: the flagged outlier `update-config` is 894 out-tok/call but 337K cache_read/call ‚Äî the default "shorten replies" advice would have been wrong. Per-call values are derivable (`tok/calls`), so only absolutes are emitted. New `meta.pricing` documents the flat-Sonnet-rate caveat (absolute USD approximate, cross-model rankings distorted until per-model pricing ships).
- **`claudii gc --insights DAYS [--yes]`** ‚Äî opt-in pruning of orphaned insights caches (sessions whose source transcript Claude Code already deleted) older than DAYS. Dry-run by default; `--yes` deletes. Orphans are kept by design (they are the only cost history beyond CC's transcript retention), so nothing runs without the explicit flag. Caches whose transcript still exists are never touched ‚Äî deleting one would not trigger re-aggregation (the marker-based scan only sees files newer than `.last-scan`). Age = `last_seen`, falling back to file mtime for degenerate caches without timestamped records. Core lives in `bin/claudii-insights gc --older-than DAYS [--yes]`.

### Changed
- **Default displays are token-first; `$` lives only in `claudii cost`.** The token-focus shift now reaches every default view. The in-session **CC-Statusline** promotes the token segment (‚Üë‚Üì) to the value colour (cyan) and dims `$cost`. The `session-*` cache gained a cumulative **`tok=`** (input+output) written by `bin/claudii-cc-statusline`, so the shell **Session Dashboard** and **`claudii se`/`si`** show token throughput + cache-hit% in place of the dollar amount (the cache schema add is backward-compatible ‚Äî old caches without `tok=` simply omit it). The bare-**`claudii` overview** account line shows `N tok today` (in+output deltas from history, same `attr_delta` attribution as `cost`) instead of `$X today`. No default view shows cost anymore ‚Äî the full dollar breakdown is `claudii cost`.
- **Shared `lib/fmt.awk` + `lib/render.sh` renderers** ‚Äî the token/large-number formatter `fmt_tok` was duplicated inline in `lib/cmd/cost.sh` and `lib/trends.awk` (awk) and mirrored as `_insights_short_num` in `lib/cmd/insights.sh` (a per-call `awk` fork). The awk copies now live in `lib/fmt.awk` (`fmt_tok`/`fmt_usd`/`rep`/`bar`/`bar_filled`, loaded via `-f` in trends and string-injection in cost); the bash side moved to `lib/render.sh` as pure-bash `_fmt_tok` (no fork) plus `_cache_hit_pct`/`_bar`/`_bar_filled`. `fmt_usd` formats thousands separators with pure integer/string math (immune to `LC_NUMERIC`). Foundation for the token-usage display shift (token throughput as the primary metric across all views, `$` demoted to `claudii cost`); verified byte-identical cost/cache output, green under `/bin/bash` 3.2.
- **`claudii trends` is token-primary.** The daily chart now leads with token throughput (per-day bars normalized to the busiest day) and demotes `$` to a dim side column; the 30-day model split is by tokens with bars instead of session-share text; the footer shows the busiest day, average API duration per session, and a 7d-vs-30d token trend. Fixes two bugs surfaced on real history: (1) the API-duration line summed the session-**cumulative** `total_api_duration_ms` column on every render row ‚Üí a ~6651h/day nonsense figure; it now delta-attributes per session (shared `attr_delta`) and reports avg minutes/session. (2) a blank/`<synthetic>` model leaked an empty `0%` slice into the model split ‚Äî now filtered, along with sub-1% noise (leaked fixture/local models). `trends --json` keeps its shape (sessions/percent retained) and gains a per-model `tokens` field.
- **`claudii cost` redesign ‚Äî the curated dollar view.** Today/Week are now share bars sorted by amount (descending) with the period's token total in the header; Months/Years render as a column matrix (model tiers √ó periods, `‚îÇ`/`‚îº` separators, no outer frame) with row and column totals, replacing the side-by-side tile renderer. Fixes three bugs found on real data: (3) Today/Week were iterated in hash order (Haiku `$0.30` printed before Sonnet `$1.16`) ‚Äî now sorted by amount; (4) leaked local/oMLX and test-fixture models polluted the legend and totals ‚Äî the view is now scoped to the canonical Claude tiers (Opus/Sonnet/Haiku/Fable); (5) the pretty output never showed the tokens it already aggregated ‚Äî Today/Week headers now carry `$total ¬∑ N tok`. `cost --json`/`--tsv` are unchanged (full data, all models). Amounts use the new locale-immune `fmt_usd` thousands formatting.
- **Incident indicator is now scope-aware and synced across all three consumers.** When a status incident exists but none of the tracked models are actually affected, ClaudeStatus (RPROMPT), CC-Statusline, and the bare-`claudii` overview show a neutral note glyph `‚ìò` (new `CLAUDII_SYM_NOTE`, dim) instead of the stage-colored ‚Äº/‚öê/‚óé alarm ‚Äî "there's a status note, your models are fine." A stage-colored icon now reliably means a tracked model really is degraded/down. The overview's ClaudeStatus line gained the indicator (it had none before); all three renderers derive it from the same `status-models` cache keys (`_incident=`, per-model state) so they can't drift.
- **Auto colors: piped output is plain text** (git-style). Pretty renderers keyed colors on nothing ‚Äî `claudii se | grep ‚Ä¶` or redirecting to a file dumped raw ANSI escapes. `bin/claudii` now blanks all `CLAUDII_CLR_*` constants when stdout is not a terminal; the theme loader skips repainting via an empty-RESET guard. `CLAUDII_FORCE_COLOR=1` opts back in (pagers: `claudii se | less -R`). `--json`/`--tsv` were already plain; the zsh prompt layers are untouched.
- **Model‚Üítier mapping consolidated + big jq programs moved to files.** The byte-identical jq `tier()` defs (√ó2 in `lib/cmd/skills-cost.sh`) became the shared module `lib/tier.jq`; the duplicated awk tier-collapse regex chains in `lib/cmd/cost.sh` / `lib/cmd/display.sh` became `lib/model_tier.awk` (`tier_label()`). The ~45-line pricing programs in skills-cost and the ~70-line merge program in `bin/claudii-insights` now live in `lib/skills-cost-rows.jq`, `lib/skills-cost-compare.jq` and `lib/insights-merge.jq` (per the "jq >3 lines belongs in a .jq file" rule; `lib/insights.jq` was the existing pattern). A new model tier now needs: `lib/tier.jq`, `lib/model_tier.awk`, the `_rates` table, `_insights_model_label()` ‚Äî down from five scattered copies. Verified byte-identical `skills-cost`/`--mcp`/`--compare`/`merge`/`cost`/`trends` output on frozen real caches.
- **`claudii skills-cost` rejects unknown flags** with an actionable error instead of silently ignoring them (`--foo` was a no-op; a typo like `--day 7` silently billed 30 days). The dispatcher now passes only the actual flags to the parser. Same treatment for `claudii-insights merge`, and a failing jq pricing program now surfaces its real error instead of masquerading as "No skill attribution data yet".
- **`claudii-insights merge --project` removed** ‚Äî the flag was parsed-but-dead since the first merge version (per-session caches carry no project field, so it never filtered anything) yet advertised in `--help` as functional. Implementing it for real needs a schema bump; until then passing it errors instead of silently no-op'ing.
- **`lib/tier.jq` is word-anchored** like the awk sibling (`(^|[^a-z])opus([^a-z]|$)` instead of bare substring) ‚Äî the two documented-in-sync tier mappers disagreed on glued names like `myopusx`. No real model id is affected (all have separators); verified byte-identical skills-cost output.
- **Shared `lib/attribution.awk`** ‚Äî the per-session baseline-delta heuristic (cost/token increments, >50%-drop = compaction reset) lived as three drifting copies in `lib/cmd/cost.sh`, `lib/trends.awk` and the overview's today-cost pass (trends' "keep both in sync" comment already undercounted them ‚Äî and pointed at the wrong file). All three now call the shared `attr_delta()`; verified byte-identical `cost --json` / `trends --json` / overview output on real history before/after.
- **CC-Statusline render-tick hygiene** (`bin/claudii-cc-statusline`): the `ps -o comm=` parent probe (cc-insomnii double-wrap detection) now only forks when insomnii delegation is active at all; the jq field separators are written as `\u001f`/`\u001e` escapes instead of raw control bytes (one place already used the escape ‚Äî raw bytes are invisible in editors/diffs and repeatedly read as `join("")` bugs in review).
- **Outlier flag rule: ‚â•2√ó median with ‚â•10 calls** (was ‚â•3√ó median, no calls floor). The 3√ó threshold never fired on real data (max observed 2.1√ó); the calls floor keeps rarely-used skills from tripping the flag on noise. JSON output documents the rule in `meta.outlier_rule`.

### Fixed
- **CLI flag/arg handling hardened (function audit).** A pass over every command surfaced a class of silent failures and one locale-dependent render bug. A **value-flag as the last argument** (`skills-cost --compare`, `--days`; `cache`/`tokens`/`tools`/`limits --days`) consumed `$@`, then the parser's trailing `shift` ran at `$#=0` and exited 1 under `set -e` ‚Äî a no-output crash; the final shift is now guarded (`shift || break`) and `skills-cost --compare` with no window prints an actionable message instead of exiting silently. `config get`/`set`/`import` with **no argument** leaked a raw bash `config.sh: line N: 3: ‚Ä¶` parameter error (from `${3:?‚Ä¶}`) ahead of the Usage text ‚Äî replaced with explicit guards that print only the clean Usage line. `claudii explain`'s separator rules used `printf '%.56s'` on a multibyte `‚îÄ` string, which truncates at **byte** 56 (mid-codepoint ‚Üí garbled glyph) under `LC_ALL=C` ‚Äî the locale the CI matrix runs the suite in ‚Äî now built char-aware via `_rep '‚îÄ' 56`. And `bin/claudii-cc-statusline` + the overview now numeric-guard the rate-limit **reset arithmetic** (`(( resets_at - now ))`): a non-numeric `resets_at` (CC format drift / ISO) drops the `‚Ü∫` countdown instead of leaking a "value too great for base" error to stderr (real payloads carry an epoch int, so normal rendering is unchanged).
- **`claudii cost --tsv` month/year rows carry their period key.** The `period` column emitted the literal `month`/`year` for every monthly and yearly row, so two distinct months (`2026-05`, `2026-06`) collapsed into indistinguishable `month‚á•Opus‚á•‚Ä¶` lines ‚Äî a TSV consumer couldn't tell them apart. The column now carries the `YYYY-MM` / `YYYY` key (today/week/alltime stay scope labels, mirroring the already-correct `--json` grouping).
- **awk number handling is locale-safe across every command (comma-decimal locales).** Under a comma locale (`LC_ALL=de_DE.UTF-8`) `/usr/bin/awk` (onetrueawk) mishandled numbers two ways: it **truncated at the radix on parse** (`"12.34"+0 == 12`, stopping at the dot) *and* printed `%.Nf` with a comma (`"12,34"`). The earlier fix guarded only the json/tsv *formatter* of `cost`/`trends`, so pretty output and all internal math still silently truncated, and several commands were missed entirely. Swept the whole class:
  - **`cost` / `trends` / `cost --forecast`** now run **both** awk stages under `LC_ALL=C` unconditionally. Stage 1 parses the history cost (`$3`), so a json-only guard still left pretty totals and the forecast burn slope corrupted (truncated, not just comma-formatted). The pretty branch is unaffected because it emits UTF-8 box chars via octal escapes and `length()` is byte-based on onetrueawk/mawk ‚Äî verified byte-identical C vs de_DE pretty output.
  - **`skills-cost`** ‚Äî every per-model price, the median (`sort -n` + awk), the 2√ó-median outlier threshold and both outlier compares, and the `--compare` Œî arrows now run under `LC_ALL=C`. This was the previously-deferred item: a Haiku skill's `$0.0123` avg parsed as `0`, distorting the cross-skill ranking the outlier rule (and session-close Phase 2.7) compares over.
  - **`fmt_tok`** (`lib/fmt.awk`) ‚Äî the token short form (`5.2M`, `2.3B`) used `sprintf("%.1fM", ‚Ä¶)` and rendered `5,2M` under de_DE, breaking the awk==bash parity with `_fmt_tok`. Rewritten to pure integer/string math, immune by construction (mirrors `_fmt_tok`'s tenths rounding).
  - **`claudii cache` / `tools`** cache-hit-rate and error-rate percentages.
  - **CC-Statusline** burn-ETA / pace / auto-compact-scale awks and the `$cost` segment, the **session-end** notification cost, and the **omlx** GB segment moved from `LC_NUMERIC=C` to `LC_ALL=C` ‚Äî `LC_NUMERIC=C` is silently overridden when the user exports `LC_ALL`, so those `printf '%.2f'` guards never fired for a real comma-locale user.

  CI now runs the full suite a second time under `LC_ALL=de_DE.UTF-8` on macOS (the strictest awk), so every value-asserting test doubles as a comma-locale guard; the suite's own float-compare helper awks were pinned to `LC_ALL=C`. Green under both locales on `/bin/bash` 3.2 and Homebrew bash 5.x.
- **CI macos-latest no longer fails the whole suite on `test_skills_cost.sh`** (23 red). The fixture helper resolved omitted `mcp`/`models` args with an inline-brace default (`${4:-{\}}`) ‚Äî macOS `/bin/bash` 3.2, which CI runs, keeps the backslash (`{\}`) where homebrew bash 5.x strips it (`{}`), so jq got invalid JSON, the fixture came out empty, and skills-cost fell through to the no-data path. Green locally (5.x), red on CI (3.2). Defaults now resolve as plain quoted literals, plus a guard re-runs the real helper under `/bin/bash` (via `declare -f`) so a 3.2-only regression fails locally too. Also `--`-terminated the `assert_contains`/`assert_not_contains`/`assert_matches` greps so a needle starting with `-` (e.g. `-it-4bit`) is a pattern, not BSD-grep flags (`grep: invalid option -- t` noise).
- **`claudii status` incident timestamps no longer truncate the timezone abbreviation.** The incident-update render reused the raw-ISO fallback's `T`‚Üíspace swap on the already-formatted `_ABS_FMT`, eating the `T` in the zone name ‚Äî `CEST` printed as `CES`, `CET` as `CE`, `GMT` as `GM`. The swap now runs only on the raw-ISO fallback branch (`_fmt_abs` failed); the formatted `YYYY-MM-DD HH:MM ZONE` string is used verbatim. The "Recent changes" / `--history` paths were never affected (they print `_ABS_FMT` directly).
- **Untracked-model incidents no longer paint every tracked model amber.** A status.claude.com incident scoped to models claudii doesn't surface (the live "access suspended for Claude Mythos 5 / Fable 5" note) lists the `Claude API (api.anthropic.com)` component because model access flows through it ‚Äî the broad-API heuristic in `bin/claudii-status` then cascaded `degraded` onto Opus/Sonnet/Haiku even though none were affected. The writer now detects when an incident names a specific Claude model family (tracked **or** untracked, via `_KNOWN_MODEL_FAMILIES`) and suppresses the broad-API fallback in that case; the unaffected models stay `ok` while `_incident=` is still persisted so the indicator surfaces. A genuine plane-wide API outage that names no model still cascades as before.
- **Project paths render `~/‚Ä¶` instead of `\~/‚Ä¶`.** Six `${var/#$HOME/\~}` substitutions (session rows, spinner labels) printed a literal backslash before the tilde ‚Äî bash keeps the backslash in a replacement string; the unescaped `~` inside a double-quoted expansion was always safe from tilde-expansion.
- **`claudii pin/unpin` no longer false-matches a session cache that lacks a `session_id=` line** ‚Äî the match variable now resets per file instead of inheriting the previous file's id.
- **`claudii explain` recognizes wrapper-chain statusLine configs.** The CC-Statusline status read `.statusLine` (a jq object dump) and substring-matched `*claudii*` ‚Äî the exact string-matching pattern the project rule bans. It now reads `.statusLine.command` and resolves through `_cc_statusline_connected`, so a wrapper chain like `cc-insomnii --after=<user-wrap>` reports `active` instead of `not configured`.
- **`claudii-insights aggregate` no longer force-rebuilds every run once orphaned caches exist.** The schema gate took the minimum `schema_version` across all cache files ‚Äî caches whose source JSONL Claude Code already deleted could never be upgraded, so every run since the v3 bump rebuilt all live sessions (~390 orphans √ó every invocation). The gate now compares a `.schema` marker written after the last successful rebuild; orphaned caches keep contributing history and merge tolerates their older shape.
